## Diskusi Solusi "Message Too Long" di Telegram OpenClaw (6 April 2026, ~10:19-10:30 UTC)

### Konteks
- **User**: Aris Setiawan bertanya: "Solusi message to long di openclaw telegram gmna ya?"
- **Batasan Telegram**: Maksimal 4096 karakter per pesan teks.
- **Masalah**: Output panjang dari exec commands, log, atau response agent terpotong/gagal kirim.

### Solusi yang Direkomendasikan Santa

#### 1. **Auto-Chunking (Pemotongan Otomatis)**
   - Split teks menjadi chunk ~4000 karakter.
   - Contoh script:
     ```bash
     echo "$output" | fold -w 4000 | while read chunk; do
       openclaw agent --agent agent1 --message "$chunk" 2>/dev/null
       sleep 0.5
     done
     ```

#### 2. **Kirim sebagai File (Send as Document)**
   - Simpan output ke file `.txt`, lalu kirim via script `telegram-send.sh`.
   - Contoh:
     ```bash
     your-command > /tmp/output.txt
     /root/.openclaw/workspace/scripts/telegram-send.sh /tmp/output.txt "Hasil output"
     ```

#### 3. **Summarize Sebelum Kirim**
   - Ringkas output panjang menggunakan skill `summarize` yang sudah terinstall.
   - Ambil hanya bagian penting (head + tail).

#### 4. **Konfigurasi OpenClaw Gateway**
   - Edit `~/.openclaw/openclaw.json`:
     ```json
     "telegram": {
       "maxMessageLength": 4000,
       "splitLongMessages": true,
       "sendLongAsFile": true
     }
     ```
   - Restart gateway setelah konfigurasi.

#### 5. **Truncate dengan Ellipsis**
   - Potong teks dan tambahkan indikator `... [truncated, N chars total]`.

#### 6. **Gunakan External Storage**
   - Upload ke pastebin/service (termbin.com) dan kirim link.

### Rekomendasi Implementasi

**Untuk penggunaan sehari-hari:**
- **Output exec/log panjang** → Kirim sebagai file (paling aman)
- **Chat response panjang** → Auto-chunking (otomatis split)
- **Artikel/dokumen** → Summarize dulu baru kirim

**Contoh wrapper cepat:**
```bash
long_exec() {
  output=$(bash -c "$@" 2>&1)
  if [ ${#output} -gt 4000 ]; then
    echo "$output" > /tmp/exec_$(date +%s).txt
    echo "📁 Output terlalu panjang, simpan sebagai file"
    ls -la /tmp/exec_*.txt | tail -1
  else
    echo "$output"
  fi
}
```

### Issue Penting: Konflik Pin Mapping
- **Default pin I2C (D1/D2)** bentrok dengan **pin relay yang diusulkan (D1/D2)**
- **Solusi yang ditawarkan**:
  1. **Opsi 1**: Ganti pin relay ke D3 (GPIO0) dan D5 (GPIO14)
  2. **Opsi 2**: Ganti pin I2C ke D6 (GPIO12) dan D7 (GPIO13) jika OLED shield support
  3. **Opsi 3**: Pakai pin berbeda untuk masing-masing komponen

### Fitur Script Wemos D1 Mini
- **WiFi Connection**: Auto connect dengan konfigurasi SSID/password
- **Sensor Reading**: Baca DHT11 setiap interval (default 5 detik)
- **Relay Control**: Kontrol 2 relay secara independen
- **OLED Display**: Tampilkan suhu, kelembapan, status relay, IP address
- **Auto Control**: Relay 1 nyala otomatis jika suhu >30°C (contoh untuk kipas)
- **Error Handling**: Try-catch untuk setiap komponen
- **Clean Exit**: Matikan relay saat program berhenti

### Cara Deploy Ringkas
```bash
# 1. Flash MicroPython ke Wemos D1 Mini
esptool.py --port /dev/ttyUSB0 erase_flash
esptool.py --port /dev/ttyUSB0 write_flash 0 esp8266-xxx.bin

# 2. Upload file ke board
ampy --port /dev/ttyUSB0 put wemos_d1_mini_iot.py main.py
ampy --port /dev/ttyUSB0 put ssd1306.py

# 3. Edit config (WiFi, pin mapping) di main.py
# 4. Reset board → jalan otomatis
```
